# 图片识别习题功能规格文档

## 1. 背景与目标

### 1.1 问题描述
当前自主习题功能只能处理纯文本格式的试卷，但小学生习题（如数学看图应用题、语文看图写话等）往往图文并茂，需要依赖图片才能完整表达题意。现有系统无法识别和处理这类习题。

### 1.2 解决方案
支持家长通过拍照方式上传习题图片，利用多模态AI模型自动识别图片中的习题内容，转化为结构化数据后纳入现有习题练习系统。

### 1.3 设计原则
- **保持主LLM稳定**：现有MiniMax对话功能保持不变
- **按需调用多模态模型**：仅在图片识别时调用硅基流动API
- **本地优先存储**：图片数据存储于IndexedDB，充分利用浏览器本地存储能力
- **内存及时释放**：识别完成后立即释放临时Base64内存
- **可扩展性**：预留未来支持更多图片格式和多模态模型的能力

## 2. 功能需求

### 2.1 核心功能

#### F1: 图片上传
- 支持拍照上传（调用设备摄像头）
- 支持从相册选择已有图片
- 支持多张图片批量上传（整卷拍摄场景）
  - 单次最多上传图片数量：5张
  - 单张图片压缩后目标大小：≤2MB（前端自动压缩）
- 支持的图片格式：JPEG, PNG, WebP
- 单张图片大小限制：10MB（压缩前）
- 图片上传前进行格式校验

#### F2: 图片识别
- 调用硅基流动API（Qwen/Qwen2-VL-72B-Instruct模型）
- 识别图片中的习题内容
- 支持多题型识别：选择题、填空题、简答题、口算题、应用题
- 支持图文混合习题（"如图所示..."类习题）
- 支持一张图包含多道习题的分割识别
- 返回结构化JSON数据

#### F3: 习题解析结果展示
- 展示识别出的习题数量和类型分布
- 展示每道习题的预览（题型、习题摘要）
- 家长可确认或取消上传
- 识别失败时显示错误原因

#### F4: 习题存储
- 结构化习题数据存入exercises.json（扩展字段）
- 原始图片数据存入IndexedDB
- 建立习题与图片的关联关系
- 支持后续练习时展示图片

#### F5: 图片在练习中的展示
- 练习界面支持显示习题关联的图片
- 图片可放大查看
- 图片位置与习题文字合理布局

#### F6: 内存管理与清理
- 图片Base64数据在识别完成后立即释放
- 已完成练习超过7天的图片数据自动清理
- 存储空间超过50MB时自动清理最旧的已使用图片
- 清理时机：每日总结生成时触发

### 2.2 用户交互流程

```
┌─────────────────────────────────────────────────────────────────┐
│                    家长上传习题流程                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 家长进入"习题上传"页面                                       │

│  7. 识别完成后展示解析结果预览                                    │
│  8. 家长确认后点击"确认上传"                                    │

```

### 2.3 边缘场景处理

| 场景 | 处理方式 |
|------|----------|
| 图片模糊无法识别 | 显示"图片模糊，请重新拍摄"提示 |
| 图片中无有效习题 | 显示"未检测到有效习题，请上传清晰的习题图片" |
| 网络请求失败 | 显示"网络错误，请检查网络连接后重试" |
| API调用超时（>30s） | 显示"识别超时，请稍后重试" |
| 识别结果为空 | 显示"未能识别到习题内容"并展示原图供家长确认 |
| 存储空间不足 | 触发清理机制，删除最旧的已使用图片 |

### 3.2 数据结构扩展

#### Exercise 接口扩展
```typescript
interface Exercise {
  id: string
  type: ExerciseType
  question: string
  options?: string[]
  answer: string
  chapter?: string
  subject?: string

  // 新增字段
  images?: string[]           // 关联的图片ID数组
  hasImageReference?: boolean // 习题是否包含"[图片]"引用标记
}
```

#### ExerciseImage 接口（新增）
```typescript
interface ExerciseImage {
  id: string               // 唯一标识(UUID)
  exerciseIds: string[]    // 关联的习题ID数组（一道习题可能关联多张图片）
  base64: string           // Base64编码的图片数据
  thumbnail?: string       // 缩略图(可选,用于预览)
  createdAt: string        // 创建时间(ISO格式)
  status: 'pending' | 'ready' | 'used'  // 状态
  size: number             // 图片大小(字节)
}
```

### 3.3 API调用

#### 硅基流动API
- **端点**: `https://api.siliconflow.cn/v1/chat/completions`
- **模型**: `Qwen/Qwen3.5-397B-A17B`
- **认证**: Bearer Token
- **请求方式**: POST
- **超时时间**: 30秒

#### 请求格式
```json
{
  "model": "Qwen/Qwen3.5-397B-A17B",
  "messages": [
    {
      "role": "user",
      "content": [
        {
          "type": "image_url",
          "image_url": {
            "url": "data:image/jpeg;base64,xxxxx"
          }
        },
        {
          "type": "text",
          "text": "[系统提示词]"
        }
      ]
    }
  ],
  "max_tokens": 8192,
  "stream": false
}
```

#### 响应格式（成功）
```json
{
  "id": "chatcmpl-xxx",
  "choices": [{
    "message": {
      "role": "assistant",
      "content": "{\"exercises\": [...]}"
    },
    "finish_reason": "stop"
  }],
  "usage": {
    "prompt_tokens": 1234,
    "completion_tokens": 567,
    "total_tokens": 1801
  }
}
```

#### 错误码处理
| HTTP状态码 | 含义 | 处理方式 |
|-----------|------|----------|
| 400 | 请求格式错误 | 检查图片格式，提示用户重新上传 |
| 401 | API Key无效 | 检查配置，提示管理员检查API Key |
| 429 | 请求频率限制 | 等待后重试，提示用户稍后再试 |
| 500/502/503 | 服务端错误 | 提示用户稍后重试 |

#### CORS说明
- 硅基流动API支持跨域请求（CORS）
- 需在实现时验证：发送OPTIONS预检请求是否正常
- 备选方案：如遇CORS问题，可使用Cloudflare Workers等边缘函数代理

### 3.4 配置扩展

在 `config.json` 中新增多模态识别配置：
```json
{
  "vision": {
    "provider": "siliconflow",
    "apiKey": "your-siliconflow-api-key",
    "baseUrl": "https://api.siliconflow.cn/v1",
    "model": "Qwen/Qwen3.5-397B-A17B",
    "timeout": 30000,
    "maxRetries": 2
  }
}
```

### 3.5 内存管理机制

```
【阶段1: 上传识别】
图片File → 转Base64 → 发送API → 收到响应 → temp变量=null

【阶段2: 存储】
响应JSON → exercises.json
图片Base64 → IndexedDB(images表)

【阶段3: 练习使用】
练习时通过imageId从IndexedDB加载图片显示

【阶段4: 清理触发】
检查已完成练习的图片
标记 status='used' 的图片
删除 createdAt > 7天 的图片
若总大小 > 50MB，删除最旧的已使用图片
```

#### 清理触发时机
清理逻辑在以下场景触发：
1. **每日总结生成时**：memory-archive.ts 的 generateSummary() 方法执行时
2. **应用启动时**：检测存储空间，若超过阈值立即清理
3. **上传新图片前**：检测存储空间，不足时先清理

#### 清理规则细化
- 优先删除：status='used' 且 createdAt > 7天 的图片
- 次优先删除：status='used' 且最旧的图片（按createdAt排序）
- 不删除：status='pending' 或 status='ready' 的图片（正在使用中）

## 8. 风险评估与应对

### 8.1 技术风险
| 风险 | 影响 | 概率 | 应对方案 |
|------|------|------|----------|
| API Key泄露 | 安全风险 | 中 | 配置文件不提交到版本控制，使用环境变量或运行时注入 |
| CORS限制 | API调用失败 | 低 | 硅基流动支持CORS，需验证；备选方案：使用代理服务 |
| 浏览器IndexedDB兼容性 | 存储失败 | 低 | 检测浏览器支持，不支持时降级为提示用户 |
| 大图片上传性能问题 | 用户体验差 | 中 | 前端压缩图片至合理尺寸（如宽度≤1920px） |
| IndexedDB存储空间不足 | 数据丢失 | 低 | 实现自动清理机制，监控存储使用量 |

### 8.2 业务风险
| 风险 | 影响 | 概率 | 应对方案 |
|------|------|------|----------|
| AI识别结果不准确 | 习题错误 | 中 | 提供家长编辑修正功能（后续迭代） |
| 硅基流动服务不可用 | 功能中断 | 低 | 提供友好的错误提示，建议稍后重试 |
| 识别费用超预算 | 成本风险 | 低 | 监控API调用量，设置预算告警 |
