# V1.2版本 - 家长管理、儿童任务与自主记忆系统规格文档

## Why

当前V1.1版本已完成基础的AI学伴对话功能，但缺乏家长管理端和儿童学习任务管理系统。V1.2版本旨在构建完整的家庭学习辅助系统，包括家长端任务管理、儿童端待办事项系统、自主习题练习功能，以及智能记忆系统，实现个性化的学习陪伴体验。

## What Changes

### 新增功能模块

#### 家长页面模块
- **每日学习任务布置功能**：提供直观的任务创建界面，支持设置任务内容、时间等
- **自主习题上传功能**：支持上传文本文件，智能识别文档结构层级，转化为结构化习题数据

#### 儿童页面功能增强
- **待办事项生成与展示系统**：
  - 根据家长布置的任务自动生成结构化待办列表
  - 直观的todolist界面，清晰展示任务名称、完成情况
  - Agent驱动的任务管理机制（非手动管理）
  - 进度可视化功能

- **自主习题练习系统**：
  - 习题自动整合到待办事项
  - 点击待办事项触发练习模式
  - 交互式问答对话系统（agent提问-用户回答-agent批改）
  - 支持多种题型（选择题、填空题、简答题）
  - 练习完成后生成得分、错题分析、学习建议

#### 自主记忆功能模块
- **对话内容总结机制**：每次对话结束自动生成结构化摘要
- **记忆文件管理系统**：
  - 设计记忆数据存储格式（待办进度、学习情况、练习完成情况）
  - 自动保存与更新记忆文件
- **状态感知机制**：对话开始前自动读取记忆文件，agent掌握用户学习状态

### 技术架构调整
- 新增家长端路由和页面组件
- 新增儿童端待办事项组件和习题练习组件
- 新增记忆系统服务模块
- 扩展数据存储结构，支持任务、习题、记忆数据

## ADDED Requirements

### Requirement: 家长页面 - 每日学习任务布置功能

系统应提供直观的任务创建界面，让家长能够轻松布置每日学习任务。

#### Scenario: 创建学习任务
- **WHEN** 家长在任务创建界面填写任务信息
- **THEN** 系统提供以下输入字段：
  - 任务名称（必填，最多50字符）
  - 任务描述（可选，最多200字符）
  - 任务类型（下拉选择：学习、练习、阅读、其他）
  - 预计完成时间（分钟，必填）
  - 任务日期（默认今天，可选未来7天）
- **AND** 点击"添加任务"按钮后，任务保存到任务列表
- **AND** 显示成功提示"任务已添加"

#### Scenario: 查看任务列表
- **WHEN** 家长进入任务管理页面
- **THEN** 系统显示所有已创建的任务列表
- **AND** 每个任务显示：名称、类型、预计时间、状态（待完成/已完成）
- **AND** 支持按日期筛选任务
- **AND** 支持删除任务（需二次确认）

#### Scenario: 编辑任务
- **WHEN** 家长点击任务列表中的"编辑"按钮
- **THEN** 系统打开任务编辑表单，预填充当前任务信息
- **AND** 修改后点击"保存"更新任务
- **AND** 显示成功提示"任务已更新"

### Requirement: 家长页面 - 自主习题上传功能

系统应支持上传文本文件，智能识别文档结构，转化为结构化习题数据。

#### Scenario: 上传习题文件
- **WHEN** 家长点击"上传习题"按钮
- **THEN** 系统打开文件选择对话框
- **AND** 支持的文件格式：.txt, .md
- **AND** 文件大小限制：最大5MB
- **AND** 选择文件后自动开始解析

#### Scenario: 习题文档结构识别
- **WHEN** 系统解析上传的习题文档
- **THEN** 系统识别以下结构层级：
  - 章节标题（以"#"或"第X章"开头）
  - 题目编号（以"1."、"一、"、"（1）"等格式）
  - 题型标识（选择题、填空题、简答题等关键词）
  - 题目内容
  - 答案（以"答案："、"答："开头）
- **AND** 将识别的内容转化为结构化JSON数据
- **AND** 显示解析预览，包含识别的题目数量和类型分布

#### Scenario: 确认上传习题
- **WHEN** 家长查看解析预览后点击"确认上传"
- **THEN** 系统保存习题数据到习题库
- **AND** 显示成功提示"已上传X道习题"
- **AND** 习题自动关联到当日待办事项

#### Scenario: 解析失败处理
- **WHEN** 系统无法识别文档结构
- **THEN** 显示错误提示"文档格式无法识别，请检查格式"
- **AND** 提供格式示例说明
- **AND** 允许重新上传

### Requirement: 儿童页面 - 待办事项生成与展示系统

系统应根据家长布置的任务自动生成待办事项，并提供直观的展示界面。

#### Scenario: 自动生成待办事项
- **WHEN** 儿童用户登录系统或进入任务页面
- **THEN** 系统自动加载当日所有待办任务
- **AND** 将任务转化为待办事项列表
- **AND** 每个待办事项包含：任务名称、类型图标、预计时间、完成状态

#### Scenario: 待办事项界面展示
- **WHEN** 儿童用户查看待办事项列表
- **THEN** 界面采用儿童友好设计：
  - 大尺寸卡片式布局
  - 鲜艳的颜色区分任务类型
  - 清晰的完成状态图标（✓已完成，○未完成）
  - 进度条显示整体完成率
- **AND** 未完成任务排在前面
- **AND** 已完成任务显示完成时间

#### Scenario: Agent驱动的任务管理
- **WHEN** 儿童用户完成一项任务
- **THEN** Agent主动询问"你完成了[任务名称]吗？"
- **AND** 用户确认后，Agent更新任务状态为"已完成"
- **AND** Agent给予鼓励性反馈（如"太棒了！你完成了今天的第一个任务！"）
- **AND** 系统自动更新进度可视化

#### Scenario: 进度可视化
- **WHEN** 儿童用户查看任务进度
- **THEN** 系统显示：
  - 整体进度条（已完成/总任务数）
  - 今日学习时长统计
  - 连续完成任务天数
  - 鼓励性文字（如"你已经完成了今天的一半任务，继续加油！"）

### Requirement: 儿童页面 - 自主习题练习系统

系统应提供交互式习题练习功能，支持多种题型的智能问答。

#### Scenario: 进入习题练习
- **WHEN** 儿童用户点击待办事项中的习题任务
- **THEN** 系统进入习题练习模式
- **AND** Agent以语音方式说"让我们开始练习吧！"
- **AND** 显示习题概览（总题数、题型分布）

#### Scenario: 选择题练习流程
- **WHEN** Agent提问选择题
- **THEN** Agent语音读出题目和选项
- **AND** 界面显示题目和选项按钮
- **AND** 用户点击选项或语音回答
- **AND** Agent判断答案并给出反馈：
  - 正确："回答正确！你真棒！"
  - 错误："这个答案不太对哦，正确答案是X，让我们看看为什么..."

#### Scenario: 填空题练习流程
- **WHEN** Agent提问填空题
- **THEN** Agent语音读出题目（空格处停顿）
- **AND** 界面显示题目和输入框
- **AND** 用户输入或语音回答
- **AND** Agent判断答案并给出反馈

#### Scenario: 简答题练习流程
- **WHEN** Agent提问简答题
- **THEN** Agent语音读出题目
- **AND** 界面显示题目和语音输入按钮
- **AND** 用户语音回答
- **AND** Agent评估答案完整性并给出反馈

#### Scenario: 练习完成总结
- **WHEN** 所有习题练习完成
- **THEN** Agent生成练习总结：
  - 总得分（如"8/10"）
  - 错题列表及正确答案
  - 错题知识点分析
  - 学习建议（如"建议复习第三章的内容"）
- **AND** 语音播报总结内容
- **AND** 更新待办事项状态为"已完成"
- **AND** 更新记忆文件中的练习记录

### Requirement: 自主记忆功能 - 对话内容总结机制

系统应在每次对话结束时自动生成结构化摘要。

#### Scenario: 对话结束触发总结
- **WHEN** 用户与Agent的对话结束（用户离开页面或长时间无交互）
- **THEN** 系统自动调用LLM生成对话摘要
- **AND** 摘要包含：
  - 对话主题（1-2句话）
  - 关键知识点（列表形式）
  - 用户表现评估（积极/需要改进）
  - 情感状态（开心/困惑/疲惫等）

#### Scenario: 摘要数据结构
- **WHEN** 系统生成对话摘要
- **THEN** 摘要以JSON格式存储：
  ```json
  {
    "timestamp": "2026-03-19T10:30:00Z",
    "duration": 1200,
    "topic": "学习了乘法口诀",
    "keyPoints": ["掌握了2-5的乘法", "对6-9的乘法还需练习"],
    "performance": "积极",
    "emotion": "开心",
    "tasksCompleted": ["完成乘法练习题"],
    "nextSuggestions": ["继续练习6-9的乘法"]
  }
  ```

### Requirement: 自主记忆功能 - 状态感知机制

系统应在每次对话开始前自动读取并分析记忆文件，使Agent掌握用户学习状态。

#### Scenario: 对话开始前加载记忆
- **WHEN** 用户开始新对话
- **THEN** 系统自动读取 `memory.json` 文件
- **AND** 提取关键信息：
  - 今日待办任务及完成情况
  - 最近学习内容和薄弱点
  - 用户偏好和最佳学习时间
  - 连续学习天数和激励状态

#### Scenario: Agent个性化问候
- **WHEN** Agent获取记忆信息后开始对话
- **THEN** Agent根据记忆生成个性化问候：

  - 晚上："晚上好！今天你完成了所有任务，明天继续保持哦！"

#### Scenario: Agent主动提醒
- **WHEN** 记忆显示用户有未完成任务或薄弱知识点
- **THEN** Agent主动提醒：
  - "我注意到你今天还有2个任务没完成，要不要现在开始？"
  - "上次练习中，乘法口诀6-9还需要加强，我们要不要再练习一下？"

#### Scenario: 记忆驱动的学习建议
- **WHEN** Agent分析记忆数据
- **THEN** Agent提供个性化学习建议：
  - "根据你的学习记录，上午是你的最佳学习时间，建议把重要任务安排在上午哦！"
  - "你最近在数学方面进步很大，继续保持！"

## Technical Implementation Notes

### 数据存储方案
- 使用JSON文件存储任务、习题、记忆数据（轻量级，无需数据库）
- 文件位置：`apps/web/public/data/`
- 文件读写通过Vite的静态文件服务和Fetch API实现

### Agent驱动机制
- 任务状态更新：通过对话意图识别，而非手动点击
- 习题批改：LLM评估答案正确性并生成反馈
- 记忆总结：LLM生成结构化摘要

### 前端架构
- Vue 3 Composition API
- Pinia状态管理
- 响应式设计，支持移动端和桌面端

### 安全性考虑
- 家长页面可选密码保护
- 数据文件定期备份
- 敏感信息不存储在客户端
